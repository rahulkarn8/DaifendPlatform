Features & how to use

Daifend is an AI-native security platform: a Next.js console plus production-structured backends (API gateway, auth, telemetry, memory integrity, agent runtime, threat/RAG, self-healing, notifications), with optional ClickHouse, Kafka/Redpanda, NATS, Qdrant, OPA, OpenTelemetry, and gRPC for memory analyze.

Console (UI)

  • Dashboard — trust timeline, drift matrix, threat ticker, runtime posture, self-healing feed.
  • Modules — AI Memory Security, Agent Runtime, Threat Intel, Incidents, Self-Healing, Research Lab, Live Simulation, Architecture, API Integrations, Settings.
  • Telemetry — demo (Node mock) or live (Python service + Socket.IO).
  • Simulation — investor-style attack spike + healing visualization.

Backend & data plane

  • API gateway — JWT / OIDC JWKS, Redis rate limits, optional mTLS to engines, optional gRPC to memory.
  • Engines — Memory integrity (numpy + optional Qdrant), agent policy + OPA, RAG scan, threat correlate, healing workflows.
  • Persistence — PostgreSQL + Alembic; telemetry to ClickHouse; streams via NATS/Kafka.
  • Web DB access — Prisma in apps/web; GET /api/health/db when DATABASE_URL is set.

How to use — quick start

Prerequisites

Node 20+, Python 3.12+ (for engines), Docker (optional, full stack).

1) UI + demo telemetry (lightweight)

cp .env.example .env
npm install
npm run dev
npm run telemetry:demo --workspace=@daifend/web

Open http://127.0.0.1:3000. Set NEXT_PUBLIC_TELEMETRY_URL if the mock server is not on port 4001.

2) Full stack (Docker Compose)

docker compose up -d --build
URL / portPurpose
127.0.0.1:3000Web console
127.0.0.1:8080API gateway (/health)
127.0.0.1:4001Telemetry (Socket.IO)
127.0.0.1:16686Jaeger (traces)
127.0.0.1:8181OPA (agent policy)
127.0.0.1:8123ClickHouse HTTP
127.0.0.1:19092Kafka API (Redpanda)

Compose sets NEXT_PUBLIC_DAIFEND_MODE=live, engine service tokens, and MEMORY_GRPC_TARGET for analyze via gRPC. See docs/LOCAL_DEV.md and docs/PHASE3.md.

3) Database migrations (Postgres)

cd packages/daifend-core && pip install -e .
export DATABASE_URL=postgresql://daifend:daifend@localhost:5432/daifend
alembic upgrade head

4) Prisma client (web)

export DATABASE_URL=postgresql://daifend:daifend@localhost:5432/daifend
npm run db:generate --workspace=@daifend/web

Health: GET /api/health/db on the Next app.

API gateway (summary)

Base URL: http://127.0.0.1:8080 (adjust for deploy). Send X-Tenant-Id. Authenticate with Authorization: Bearer <token> from POST /v1/oauth/token, or (dev) X-Internal-Token matching INTERNAL_SERVICE_TOKEN.

POST/v1/oauth/token — obtain access token
POST/v1/memory-integrity/analyze — memory / embedding integrity (gRPC if MEMORY_GRPC_TARGET set)
POST/v1/agent-runtime/validate-action — tool / policy (+ OPA when configured)
POST/v1/rag/scan-document, /v1/threats/correlate
POST/v1/self-healing/workflows, /v1/notify

TypeScript contracts: package @daifend/types; client helper: @daifend/sdk. Full detail: docs/API.md.
