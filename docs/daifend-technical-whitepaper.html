Daifend: AI Security Infrastructure for Agents, Vector Memory, and RAG

This paper describes Daifend’s AI-native security model: how Daifend measures integrity, detects manipulation, enforces runtime policies, and restores trusted state for enterprise AI systems.

Abstract

Modern AI systems are autonomous, tool-using, and stateful. They introduce new security domains: semantic manipulation, memory poisoning, retrieval chain corruption, and policy evasion in reasoning. Daifend provides a unified control plane for detection + enforcement + recovery.

Design goals

  • Believable telemetry that captures AI-native signals (drift, lineage, tool misuse, signatures).
  • Deterministic policy for memory writes, retrieval, and tool execution.
  • Autonomous recovery with auditable rollbacks and validation gates.
  • Enterprise integration with SOC exports without becoming a generic SIEM.

Key observation

AI attacks target meaning and state. Effective defense requires a runtime control plane that can: (1) observe semantic integrity, (2) enforce policy deterministically, and (3) restore trusted state autonomously.

2) Telemetry and integrity metrics

Memory trust score (0–100)

A composited integrity score computed per memory namespace and time window.

  • Lineage confidence: provenance completeness (ingestion → embed → write).
  • Semantic consistency: drift relative to baseline embeddings.
  • Poison signatures: known payload patterns in embedding space.
  • Write anomaly: unusual burst writes, rare sources, unauthorized agents.

Drift score (0–1)

Measures stability of meaning over time. Drift correlates with stealth poisoning and context steering. 

drift = f(
  centroid_shift,
  neighborhood_entropy,
  retrieval_disagreement,
  lineage_break_rate
)

Runtime signals

  • Unsafe tool attempts: blocked tool calls, restricted command patterns, anomalous parameters.
  • Containment actions: quarantines, session rotations, rollback orchestration steps.
  • Threat signatures: RAG bypass, embedding poisoning, model manipulation, synthetic identity.

3) Policy enforcement model

Policy primitives

  • Memory write policy: required lineage, allowed sources, trust threshold, quarantine rules.
  • Retrieval policy: signed context windows, maximum contamination budget, injection filters.
  • Tool policy: risk-tiered permissions, escalation gates, irreversible action blockers.
if (memory.trust < T_low || drift > D_high) {
  retrieval.mode = "gated";
  memory.writes = "quarantine";
  agent.tools = "restricted";
}

5) Architecture integration

Where Daifend sits

  • Between apps and agents as a runtime policy and telemetry layer.
  • Alongside RAG ingestion and retrieval for scanning + chain validation.
  • With vector databases for trust scoring, lineage, and rollback operations.
  • Exports to SOC systems for alerting and incident response coordination.
